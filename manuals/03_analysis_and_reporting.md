# Analysis & Reporting — Dashboards and Tables

CoEval's `analysis/` package transforms completed experiment artifacts into
HTML dashboards, CSV tables, and paper-quality result summaries.

---

## 1. Quick Start

```bash
# Generate all HTML reports for a completed run
python -m analysis.main --run-path benchmark/runs/medium-benchmark-v1

# Generate paper tables (requires benchmark-native scores)
python -m analysis.paper_tables --run-id paper-eval-v1 --out-dir paper/tables
```

## 2. Available Report Types

### 2.1 HTML Dashboards

All HTML reports are generated by `analysis.main` and written to the run folder:

| Report file | Description | Key content |
|-------------|-------------|-------------|
| `*_student_report.html` | Per-student breakdown | Score distribution, per-task ranking, per-rubric heatmap |
| `*_teacher_report.html` | Per-teacher/source breakdown | Coverage across attribute strata, quality consistency |
| `*_judge_report.html` | Per-judge breakdown | Positional bias rates, score calibration drift |
| `*_score_dist.html` | Score distribution overview | Histograms of High/Medium/Low per rubric factor and task |
| `*_coverage.html` | Attribute coverage summary | Attribute Coverage Ratio (ACR), rare-attribute recall |
| `*_interaction.html` | Teacher×Student interaction matrix | Which (teacher, student) pairs produce highest/lowest scores |
| `*_consistency.html` | Judge consistency analysis | Inter-judge agreement (ICC), flagged high-uncertainty items |
| `*_robust_summary.html` | Robust ensemble summary | Final model rankings, confidence intervals |

**Generate all HTML reports:**
```bash
python -m analysis.main \
    --run-path benchmark/runs/medium-benchmark-v1 \
    --out-dir  benchmark/runs/medium-benchmark-v1/html_reports
```

**Generate a single report type:**
```bash
python -m analysis.main \
    --run-path benchmark/runs/medium-benchmark-v1 \
    --report   student_report
```

### 2.2 Excel Export

A single Excel workbook with one sheet per report type:

```bash
python -m analysis.main \
    --run-path benchmark/runs/medium-benchmark-v1 \
    --format excel \
    --out-file benchmark/runs/medium-benchmark-v1/analysis.xlsx
```

Sheets included:
- `Summary` — overall model rankings + composite scores
- `StudentScores` — full per-(student, task, rubric_factor) score table
- `TeacherCoverage` — attribute coverage metrics per teacher
- `JudgeAgreement` — pairwise ICC between all judge pairs
- `FailedRecords` — records with validation errors (MISSING_RESPONSE etc.)

**Automatically computed metrics (new as of 2026-03-02):**

- **RAR** (Rare-Attribute Recall) — fraction of rare strata (freq < 3) covered; Table 4 + Table 7.

## 3. Understanding the Data Model

All reports are built on top of the unified `EESDataModel` (loaded by
`analysis.loader.load_ees`). The key analytical unit is:

```
(response_id, rubric_factor) → score ∈ {High, Medium, Low}
```

Normalised to floats: `High=1.0`, `Medium=0.5`, `Low=0.0`.

**Validity classification.** A Phase 5 record is valid if:
- The referenced Phase 4 response exists (`MISSING_RESPONSE`)
- The referenced Phase 3 datapoint exists (`MISSING_DATAPOINT`)
- All rubric factors are present in the scores (`INCOMPLETE_SCORES`)
- All score values are `High`, `Medium`, or `Low` (`INVALID_SCORE_VALUE`)

Invalid records appear in the Excel `FailedRecords` sheet but are excluded from
all aggregate statistics.

**Self-judging / self-teaching flags.** Records where `judge_model_id ==
student_model_id` are flagged `is_self_judging=True`; records where
`teacher_model_id == student_model_id` are flagged `is_self_teaching=True`.
Reports can be filtered to exclude these.

## 4. Running Analysis Programmatically

```python
from analysis.loader import load_ees
from analysis.metrics import (
    composite_score_by_student,
    coverage_ratio,
    judge_consistency,
)

model = load_ees("benchmark/runs/medium-benchmark-v1", partial_ok=True)

# Print any load warnings
for w in model.load_warnings:
    print("WARN:", w)

# Student composite scores (mean normalised score across all valid units)
scores = composite_score_by_student(model)
for student, score in sorted(scores.items(), key=lambda x: -x[1]):
    print(f"  {student}: {score:.3f}")

# Attribute coverage ratio
acr = coverage_ratio(model, task_id="text_summarization")
print(f"ACR (text_summarization): {acr:.3f}")

# Judge consistency (ICC per task)
icc = judge_consistency(model)
for task, val in icc.items():
    print(f"  ICC({task}): {val:.3f}")
```

## 5. Key Metrics Reference

### 5.1 Composite Score

```
Q(student) = mean over all valid (response, factor) units of score_norm
```

where `score_norm ∈ {0.0, 0.5, 1.0}` (Low / Medium / High).

Rubric-weighted composite:
```
Q_weighted = Σ_l  w_l × mean(scores on factor l)
```

### 5.2 Attribute Coverage Ratio (ACR)

```
ACR = |{ω ∈ Ω : count(ω) ≥ 1}| / |Ω|
```

where Ω is the full attribute stratum space (Cartesian product of all
`target_attributes` values). Perfect coverage: ACR = 1.0.

### 5.3 Rare-Attribute Recall (RAR)

```
RAR = |{ω ∈ Ω_rare : count(ω) ≥ 1}| / |Ω_rare|
```

where Ω_rare contains strata with fewer than 3 natural occurrences in the
benchmark. Measures coverage of underrepresented scenarios.

### 5.4 Surface Bias

```
Surface Bias = mean pairwise BLEU between all prompt pairs
              (lower = more diverse prompts)
```

### 5.5 Positional Flip Rate

```
PFR = |comparisons where judge ranking changes on order swap| / |total comparisons|
```

Measured before and after swap-and-average mitigation.

### 5.6 Spearman ρ (benchmark validation runs only)

```
ρ = Spearman rank correlation between:
    - CoEval composite score Q for each student response
    - benchmark_native_score for the same datapoint
```

Computed at the response level across all 620 × 4 = 2,480 datapoints. Requires
benchmark-native scores to be populated (see manual 02, Section 5).

## 6. Report: Student Report

The student report is the primary model evaluation summary. For each student
model it shows:

- **Overall ranking** — composite score Q with 95% bootstrap CI
- **Per-task scores** — separate Q for each task
- **Per-rubric heatmap** — mean score per (task, rubric_factor) cell
- **Score distribution** — stacked bar: High/Medium/Low proportions
- **Self-teaching effect** — how score changes when teacher == student

```bash
python -m analysis.main --run-path benchmark/runs/medium-benchmark-v1 \
    --report student_report
```

---

## 7. Report: Coverage Summary

Shows how well the benchmark datapoints cover the attribute space.

- **ACR gauge** — overall and per-task attribute coverage ratio
- **Stratum heatmap** — grid of all target-attribute value combinations,
  coloured by datapoint count
- **Rare-attribute recall** — proportion of underrepresented strata covered
- **Surface bias** — mean pairwise BLEU across prompts

```bash
python -m analysis.main --run-path benchmark/runs/medium-benchmark-v1 \
    --report coverage
```

---

## 8. Report: Judge Consistency

Measures inter-judge agreement and calibration drift.

- **ICC matrix** — pairwise intraclass correlation between judges per task
- **Drift chart** — ICC over sliding window of 20 items (detects rubric drift)
- **Calibration parameters** — α and β per judge per task
- **High-uncertainty items** — responses where inter-judge σ > 1.5 on any factor

```bash
python -m analysis.main --run-path benchmark/runs/medium-benchmark-v1 \
    --report judge_consistency
```

---

## 9. Report: Interaction Matrix

The teacher×student interaction matrix reveals whether score differences are
driven by data source (teacher) rather than model capability (student).

- **Heatmap** — mean composite score for each (teacher, student) cell
- **Row effects** — teacher contribution to scores (should be small)
- **Column effects** — student contribution (should be large)
- **Self-teaching cells** — highlighted where teacher == student

```bash
python -m analysis.main --run-path benchmark/runs/medium-benchmark-v1 \
    --report interaction
```

---

## 10. Partial / In-Progress Runs

All analysis commands accept `--partial-ok` to run on an experiment that is
still in progress:

```bash
python -m analysis.main \
    --run-path benchmark/runs/medium-benchmark-v1 \
    --partial-ok
```

A warning banner is shown at the top of every report indicating the run is
incomplete, and any statistics are clearly marked as preliminary.

---

## 11. Comparing Multiple Runs

To compare two experiment runs side-by-side:

```bash
python -m analysis.compare \
    --run-a benchmark/runs/medium-benchmark-v1 \
    --run-b benchmark/runs/paper-eval-v1 \
    --out-dir paper/comparison
```

Generates a comparison report showing ranking differences, score delta per
student, and coverage improvement between runs.

## 12. CLI Reference

```
python -m analysis.main
  --run-path PATH       Experiment folder (required)
  --out-dir  PATH       Output directory (default: {run-path}/html_reports)
  --report   TYPE       One of: student_report, teacher_report, judge_report,
                        score_dist, coverage, interaction, consistency,
                        robust_summary, all  (default: all)
  --format   FMT        html | excel | csv  (default: html)
  --partial-ok          Allow analysis of incomplete runs
  --exclude-self-judge  Exclude self-judging records from all statistics
  --exclude-self-teach  Exclude self-teaching records from all statistics

python -m analysis.paper_tables
  --run-id   ID         Experiment ID (resolved under benchmark/runs/)
  --out-dir  PATH       Directory for CSV table files
  --tasks    LIST       Comma-separated task IDs (default: all)

python -m benchmark.emit_datapoints
  --dataset  NAME       xsum | codesearchnet | aeslc | wikitablequestions | all
  --run-id   ID         Create phase3_datapoints/ under benchmark/runs/{ID}/
  --out-dir  PATH       Override output directory
  --sample-size N       Items per dataset (default: 620)
  --split    NAME       Dataset split (default: loader default)
  --seed     INT        Sampling seed (default: 42)
```
