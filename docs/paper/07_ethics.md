# Ethics Statement

**Data sources and licensing.** All benchmark datasets used in this paper (XSum, HumanEval, CodeSearchNet, ChartQA) are publicly available under permissive research licenses. XSum is provided under the MIT License; HumanEval and CodeSearchNet are released by their respective authors for research purposes; ChartQA is available under the Creative Commons Attribution 4.0 International License. No personally identifiable information (PII) is collected, stored, or processed at any stage of the CoEval pipeline. Email composition evaluations use a reference corpus derived from the AESLC dataset [citation], which has been pre-anonymized by its publishers.

**Fictitious result data.** This draft paper contains placeholder numerical results generated to illustrate the expected structure and magnitude of experimental findings. These values are not derived from actual experiments. All tables and figures marked with "Placeholder" in the source Markdown must be replaced with real experimental results before any submission or public dissemination. Publishing fictitious experimental results as real findings would constitute scientific misconduct; the placeholder data in this draft serves only the purpose of developing and reviewing the paper's structure.

**Environmental impact.** Running the full CoEval evaluation pipeline (2,480 datapoints × 8 students × 3 judges via cloud APIs) generates an estimated 12,400 API calls across providers. We mitigate environmental impact through (1) the OpenAI and Anthropic Batch APIs, which schedule requests during off-peak periods; (2) incremental checkpointing that eliminates redundant recomputation on reruns; and (3) per-model quota limits that prevent runaway usage. Researchers deploying CoEval at larger scales are encouraged to report the compute cost in their papers and to set conservative quota limits.

**Potential for misuse.** The teacher-student-judge paradigm can, in principle, be used to mass-produce synthetic evaluation data of dubious quality if teacher models are misconfigured or if users disable calibration. To mitigate this, CoEval's robust-filter export (`coeval analyze export-benchmark`) only exports datapoints where the judge ensemble agrees above a configurable threshold, preventing the inadvertent publication of low-confidence or label-noise-contaminated benchmark items. Users who publish CoEval-generated benchmarks as standalone datasets should report the judge agreement statistics and calibration parameters alongside the data.

**Model bias amplification.** If teacher models encode systematic biases—for example, preferring culturally Western writing styles or overrepresenting STEM domains—those biases will be reflected in the generated benchmark corpus. We encourage users to audit teacher-generated datapoints for demographic, cultural, and domain diversity, particularly before using CoEval in high-stakes evaluation settings. The coverage-summary report (`coeval analyze coverage-summary`) can be used to detect systematic gaps in the attribute distribution.
