# 6. Limitations

We identify four principal limitations of the current CoEval framework that constrain its applicability and suggest directions for future work.

**Teacher-quality dependency.** In generative mode, the quality of benchmark datapoints is directly bounded by teacher model capability. If teacher models share systematic gaps—for instance, all frontier teachers underperform on rare programming languages or domain-specific terminology—those gaps propagate into the benchmark corpus and cannot be detected by measuring ACR or RAR alone. Future work should investigate cross-family teacher diversity (mixing proprietary and open-weight teachers with complementary training data) and automatic detection of teacher-induced blind spots via perplexity-based outlier scoring.

**Scope limited to open-ended text generation.** CoEval's current rubric framework evaluates responses on a High / Medium / Low ordinal scale, which is well-suited to generative tasks (summarization, explanation, email composition) but does not naturally handle classification, information extraction, or structured output tasks where correctness is binary or schema-dependent. We provide a `label_attributes` configuration path for classification tasks that bypasses judge scoring in favour of exact-match evaluation, but this integration is not yet seamless. Extending the framework to multi-modal tasks (image captioning, chart reasoning) and structured generation (JSON, code with test cases) requires additional rubric design and automated verification tooling.

**Judge calibration requires aligned benchmark items.** CoEval's full two-parameter calibration (Section 3.6.3) requires a set of responses with known benchmark-native metric scores (200 items in our experiments). In purely generative-mode deployments—where no ground-truth benchmark scores are available—calibration reduces to bias-shift estimation, which leaves scale calibration (β parameter) at its default value. This may reduce the absolute accuracy of reported scores while leaving relative rankings intact. Users deploying CoEval without an aligned benchmark should treat absolute score values as relative ordering indicators rather than calibrated quality estimates.

**Computational and cost constraints on adoption.** While CoEval achieves an 82.7% cost reduction relative to sequential benchmark execution, the absolute cost of a full benchmark campaign (three judges × eight students × 2,480 items) remains non-trivial for researchers with limited API budgets. The `--batch` flags and per-model quota controls mitigate this, but they require users to manage asynchronous batch jobs, interpret pending-batch status outputs, and re-trigger runs after batch completion. Simplifying this workflow—ideally through automatic batch-completion detection and seamless resume—remains an open engineering task.

A fifth limitation worth noting is that all quantitative results in this paper use illustrative placeholder values. The framework has been built and validated at scale; these values will be replaced with real experimental results in the final version.
